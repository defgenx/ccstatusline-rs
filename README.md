# ccstatusline

A fast, native status line for [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI, written in Rust.

This is a Rust rewrite of the original TypeScript [ccstatusline](https://github.com/sirmalloc/ccstatusline) by [@sirmalloc](https://github.com/sirmalloc). The goal is a zero-dependency, single-binary replacement that starts instantly and avoids the Node.js runtime overhead.

## What it does

`ccstatusline` reads Claude Code's JSON status data from stdin and outputs a multi-line ANSI-colored status bar:

- **Line 1** - Model name, session/daily/block costs, burn rate, and context window usage
- **Line 2** - Current working directory and git branch
- **Line 3** - Token breakdown (total, input, output, cached)

## Installation

### Build from source

Requires [Rust](https://rustup.rs/) (stable).

```sh
git clone https://github.com/sirmalloc/ccstatusline.git
cd ccstatusline
cargo build --release
cp target/release/ccstatusline ~/.claude/ccstatusline-bin
```

### Configure Claude Code

Add the following to `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "/path/to/ccstatusline-bin",
    "padding": 0
  }
}
```

Replace `/path/to/ccstatusline-bin` with the actual path (e.g. `~/.claude/ccstatusline-bin`).

## How it works

Claude Code pipes a JSON object to the command's stdin on each status update. The JSON contains fields like:

| Field | Description |
|---|---|
| `cwd` | Current working directory |
| `model` | Active model name or object |
| `cost.total_cost_usd` | Session cost in USD |
| `context_window.total_input_tokens` | Cumulative input tokens |
| `context_window.total_output_tokens` | Cumulative output tokens |
| `context_window.current_usage` | Current usage including cache stats |
| `context_window.context_window_size` | Max context window size |

The binary also scans `~/.claude/projects/` JSONL session files to compute daily and rolling block costs, with burn rate estimation.

## Origin

This project is a Rust translation of the original TypeScript implementation:
**[https://github.com/sirmalloc/ccstatusline](https://github.com/sirmalloc/ccstatusline)**

The TypeScript version uses Node.js and npm. This Rust port produces a single static binary with no runtime dependencies, resulting in faster startup and lower resource usage.

**Note:** The entire Rust translation was generated by Claude Code (Opus 4.6). While it has been tested and works, use it with care and review the code before relying on it in production.

## Dependencies

- [serde](https://crates.io/crates/serde) + [serde_json](https://crates.io/crates/serde_json) for JSON parsing
- No runtime dependencies beyond the binary itself
